# 使用说明

## 启动方式

Windows 用户双击：

```text
点我启动.vbs
```

第一次启动会自动准备 Python、Node.js、后端、平台端和移动端，可能需要几分钟；后续启动会明显变快。启动过程静默执行，不会弹出黑色命令行窗口，也不会显示启动状态小窗口。

启动成功后会自动打开：

- 平台端：默认 `http://127.0.0.1:5173`
- 移动端：默认 `http://127.0.0.1:5174`

如果默认端口被占用，系统会自动换到可用端口，并打开正确地址。

## 登录账号

- 平台端输入：`admin`
- 移动端输入：`inspector`
- 当前本地交付版不需要输入密码。

登录后，平台端用于查看数据、复核和生成报告；移动端用于现场填报和提交数据。

## 正常使用流程

1. 双击 `点我启动.vbs`，等待平台端和移动端自动打开。
2. 在移动端选择项目、年度、季度、镇街和考核对象，完成填写后提交。
3. 在平台端进入“数据看板”确认数据已经同步，再到“数据复核”完成复核。
4. 在“生成报告”中选择当前项目和周期，生成单镇报告或汇总报告。
5. 在“历史报告”中预览或下载生成的 DOCX 报告。

首次启动需要下载运行环境和依赖，请保持网络连接。若几分钟后页面仍未自动打开，再查看下方启动日志。

## 日志和故障排查

启动日志和端口记录在：

```text
排水\运行脚本\watersupply-agent-runtime\logs
```

常用文件：

- `startup-status.txt`：启动进度记录。
- `startup-status.json`：当前端口和启动状态。
- `startup.log`：详细启动日志。

如果启动失败，会弹出中文提示，并告诉你日志位置。

需要他人协助排查时，发送 `startup.log` 即可，不需要发送数据库或整个项目文件夹。

## 运行数据位置

为保持交付目录干净，运行时生成物不会写入 `watersupplyassessment` 源码目录。

默认运行数据目录：

```text
排水\运行脚本\watersupply-agent-runtime
```

这里会保存：

- 后端运行依赖
- 前端运行副本和依赖
- 本地数据库
- 上传附件和生成报告
- 启动日志

生成的正式报告保存在该目录的 `storage\generated_reports` 中，也可以直接在平台端“历史报告”页面下载。

## 停止服务

需要关闭后台服务时，双击：

```text
停止服务.vbs
```

它会停止本次启动记录的后端、平台端和移动端进程。

## 目录原则

- `watersupplyassessment\Agent` 只保留源码、锁文件、说明和入口。
- `运行脚本\watersupply-agent-runtime` 保存所有可以重新生成的运行数据。
- 不需要手动打开 `内部脚本`，它们由外层 VBS 自动调用。
