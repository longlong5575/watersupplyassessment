# 报告

## 核心规则

1. 原正文只作为模板和格式参考。里面的数据、镇情、金额、系数、问题举例和结论都默认当作不存在。
2. 数据必须来自当前项目的 `资料收集`、公共资料、金额基础数据或用户本次明确指定的文件。
3. 保留底稿的封面、目录、章节顺序、标题层级、表格样式、表头、公式说明和正式报告语气。
4. 正文和文件名都不能出现“套话保留版”“正文框架版”“合并版”“技能生成版”“高仿原文版”“专项版”等制作痕迹。
5. 生成后必须检查文件名、串镇、表格序号、字体和金额计算。

## 标准资料结构

把项目资料放在工作项目根目录下：

- `资料收集/<镇名>/<镇名>附件资料.docx`
- `资料收集/公共资料/公共资料.docx`
- `资料收集/金额基础数据.xlsx`

如果新资料包没有金额基础数据，使用本 skill 的通用金额基数：

- `assets/common_amount_basis.xlsx`

只有用户提供新的合同、补充协议或金额基础表时，才更新通用金额基数。

## 生成规则

- 单镇报告只写本镇，不出现其他镇名。
- 多镇或总报告必须写共性问题分析；跨镇例子必须来自当前资料。
- 第三章金额使用 `金额` skill 重新计算。
- 第二章文字使用 `规范` skill 的正式报告语气。
- 目录、章节编号、表格标题要与实际内容一致。

## 金额规则

- 单站评分 W 换算：
  - W >= 90：Ec1 = 1，Ec2 = 1
  - 80 <= W < 90：Ec1 = [100 - (90 - W) * 0.5]%，Ec2 = 1
  - 70 <= W < 80：Ec1 = [95 - (80 - W)]%，Ec2 = [100 - (80 - W) * 0.5]%
  - 60 <= W < 70：Ec1 = [85 - (70 - W) * 1.5]%，Ec2 = [95 - (70 - W)]%
- 镇级 Ec1/Ec2 为抽检设施算术平均值，报告通常保留三位小数。
- 可用性付费 = 年可用性付费基数（万元/年）* 10000 / 12 * E1 * Ec2。
- 第一至第七批运维服务费 = 对应月运维基数合计 * Ec1。
- 扣减金额 = 基数 - 本次考核计算金额。

## 文件命名

- 单镇：`<镇名>2023年下半年度村级设施考核报告（正文）.docx`
- 总报告：`台山市2023年下半年度村级设施考核报告（正文）.docx`

## 字体和表格

- 全部文字用宋体。
- 只有标题/标题句加粗；普通正文、计算说明、问题描述、表格内容不加粗。
- 按镇过滤或拆分表格后，序号列必须从 `1` 开始连续编号；单行数据表序号必须为 `1`。

## 检查脚本

可迁移版内置检查脚本：

- `运行脚本/排水/报告检查/validate_report_outputs.py`
- `运行脚本/排水/报告检查/sample_render_report.py`

默认先做结构检查和抽样渲染，不做慢速全文渲染。

## 批量运行

在项目根目录执行：

`python3 可迁移版/运行脚本/排水/报告生成/batch_generate_town_reports.py`

脚本默认读取项目根目录的 `资料收集`，并将17份镇街报告和1份台山市总报告写入项目根目录的 `生成`。测试时可通过 `REPORT_DATA_DIR` 和 `REPORT_OUTPUT_DIR` 指定其他输入、输出目录。
