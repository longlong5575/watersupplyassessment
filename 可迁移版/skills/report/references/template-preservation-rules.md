# Template Preservation Rules

This reference records the user's preferred report style.

## Preferred Output

The report should feel like the original正文 with only the data changed. It should not feel newly written by an assistant.

## Good Pattern

- Start from the original正文 chapter framework.
- Keep 第一章的项目背景、考核依据、考核方法、考核安排套话.
- Keep 第二章 town-section wording close to the original town section.
- Keep 第三章 payment basis, formula, caveat, and result-analysis language close to the original.
- Keep 第四章 problem/recommendation headings and stock整改 language; replace only irrelevant town examples.

## Bad Pattern

- Writing a fresh executive-style summary that changes the original voice.
- Compressing the original正文 into a short two-town memo.
- Replacing original phrases with “更自然”的概括句.
- Moving payment calculations into informal explanation.
- Removing boilerplate because it seems repetitive.

## Example User Intent

When the user says “套话都不要动，改数据和正文描述就行”, do exactly that:

- Do not polish stock language.
- Do not simplify the report into a new outline.
- Treat the old正文 as a fill-in template.
- Replace only variable facts, tables, and town-specific examples.
